Scrcpy 修改

添加自动分段录制功能(目前有很多问题未解决)
scrcpy --record-segment=30 --record=r:\Logs\ScreenRecoder[%03d].mkv

已知问题
分段时长 有时会比指定的长很多(分段时需等关键帧,否则第二段会花屏无法播放)
播放进度 第二段播放时进度会从第一段结束时间开始算(播放正常,就是播放器进度条会有问题无法拖动定位)
分段在资源管理器的显示时长有问题

2026-06-21
修复PTS DTS 解决播放进度问题与显示总时长错误问题

2026-06-22
修改Server 强制5秒请求一个关键帧控制分段时长误差到5秒内
但这样会导致输出文件体积变大一点
目前想法是在需要分段时再在client向server发送REQUEST_KEYFRAME命令请求关键帧

2026-06-24
改进分段逻辑
原逻辑每段都会比指定长度长一点,导致长时间录制时分段时间与期望分段时间偏差很大
例如 分段时长设置为30s 实际分段时长可能会在32s左右
现改为累计多出时长然后从下一段扣除多出长度
